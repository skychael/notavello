A Claude Desktop extension for Windows

Share one window. Keep control.

Koga Lens lets Claude inspect the latest frame from one Windows application that you explicitly select and approve. Capture stays local until the frame is handed to Claude Desktop.

User-approved Windows application
Only this window

The user selects the source through the Windows capture picker. Claude cannot silently choose another window, process, file, or path.

SharingLatest frame only · explicit state

Designed around consent

A narrow view, with visible boundaries.

Koga Lens exposes one parameter-free tool, GetCurrentView. The user—not Claude—chooses the approved window.

One approved window

Windows presents the capture picker and the user approves a specific application window. Switching sources requires a new approval.

Local capture

Koga Lens has no cloud relay, telemetry, or upload of its own. The current frame is processed locally before being returned through Claude Desktop.

Latest frame only

Frames older than the two-second maximum age return FRAME_STALE with no image, rather than being presented as current.

Visible state

Sharing, Paused, Stopped, SourceLost, and stale conditions are explicit so the absence of a current view is not ambiguous.

Verifiable bytes

The diagnostic panel previews the exact decoded image bytes returned in the most recent response and compares hashes across the response and audit record.

No automatic saving

The latest frame is transient and in memory. Saving the exact returned bytes requires a deliberate user action and a user-chosen path.

Two different products

Koga Lens is not Koga Browser.

They share the Koga name and Notavello home, but they solve different problems on different platforms.

Koga LensA Windows application and Claude Desktop MCP extension for inspecting one user-approved window.
Koga BrowserA separate minimalist web browser available for mobile platforms.
