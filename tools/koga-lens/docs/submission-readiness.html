Koga Lens / Docs / Submission readiness

Status as of July 20–21, 2026

Submission readiness

Capture correctness is verified on the installed 0.2.7 bundle. Directory submission still depends on release, clean-machine, manifest, annotation, documentation, and repository decisions.

Current build

  • Version: Koga Lens 0.2.7
  • Format: Windows desktop application and Claude Desktop MCP extension
  • Tool: GetCurrentView, with no parameters
  • Installed verification: completed on the development machine using real Notepad and browser windows

Distribution and directory listing are different

An MCPB bundle can be distributed directly for manual installation without a directory listing. A Claude Connectors Directory listing is a separate discovery path with submission requirements, automated policy scanning, and a potential later verified review.

This website does not offer a public download because the handoff does not provide a verified public download URL.

Verified and complete

  • Simple and dense text reads on the installed 0.2.7 bundle.
  • Frozen-frame regression checks after content changes and after switching between browser applications.
  • Stop-clear behavior and explicit no-frame state.
  • Exact returned-image preview, byte identity checks, and hash comparison across preview, response metadata, and audit entry.
  • Local capture design with one user-approved window and explicit freshness enforcement.

Submission gates still open

  1. Clean-machine installation: cold install and run on a vanilla Windows machine with Claude Desktop. This remains the highest-value unknown.
  2. First-contact STOPPED response: confirm that a reviewer who calls the tool before sharing receives useful guidance.
  3. Privacy publication: publish a stable HTTPS policy and reference it from the extension manifest.
  4. Manifest and annotations: confirm manifest version 0.2 or later, the privacy policy URL, and emitted title and readOnlyHint: true tool annotations.
  5. Public documentation: make documentation public by publish date; it may be shared privately during review.
  6. Repository decision: scan full Git history, then make the private repository public only if proceeding with directory submission. That irreversible step is last.

Recommended order

  1. Choose manual distribution or directory listing.
  2. Run clean-machine, remaining non-Notepad, and STOPPED first-contact checks.
  3. Publish privacy and add the manifest policy URL.
  4. Verify tool annotations using MCP Inspector and rebuild if needed.
  5. Complete public documentation.
  6. Scan Git history and, only for the directory path, make the repository public.
  7. Submit through the desktop-extension MCPB form with the reviewer runbook and example prompts.

Reviewer runbook summary

  1. Use Windows 10 or 11 with Claude Desktop.
  2. Install the 0.2.7 MCPB, fully quit Claude Desktop, and relaunch it.
  3. Launch the Koga Lens tray application.
  4. Open a window, select it in the tray, and activate sharing.
  5. Ask Claude to call GetCurrentView.

Calling before sharing returns STOPPED. That is expected behavior; activate sharing and retry.
